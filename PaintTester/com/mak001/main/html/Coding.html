Not sure about Graphics?

The Graphics class (java.awt.Graphics) offers methods to draw lines and shapes onto a canvas (the RuneScape game screen in this case). Many of the common ones used in paints are drawString(String, int, int), drawLine(int, int, int, int) and setColor(Color).

There are some basic classes that are used when creating a simple paint. Some of these are Color, Graphics2D (explained later), and BasicStroke.

About Graphics2D

The Graphics2D class (java.awt.Graphics2D) is like the Graphics class, but has a couple new methods. These methods can be helpful in making text and shapes smooth (called Antialiasing), giving lines and shapes' outer edges thickness(using BasicStoke), scale and rotate the entire paint (not covered in this help page, google it).

Many people automatically make the Graphics object that was passed in the onRepaint method into a Graphics2D object. This is how you do it:

public void onRepaint(Graphics g1) {
      Graphics2D g = (Graphics2D) g1;
      g.(insert method here);
}

As you can see all it that was done was casting a Graphics object to a Graphcis2D object. (If you have no clue what casting is, google it, it is very useful)

About Antialiasing

As said in the Graphics2D section, antialiasing is used to make lines, shapes and text smooth. This will only work with a Graphics2D object, meaning if you try and do this with the normal Graphics object it will give you errors.

public void onRepaint(Graphics g1) {
      Graphics2D g = (Graphics2D) g1;
      g.setRenderingHint(RenderingHints.KEY_ANTIALIASING,
           RenderingHints.VALUE_ANTIALIAS_ON);
}

You may be asking, "This must be more complicated, where is the rest?". Simply put, there is no more to it. Once the Graphics2D object has a Rendering Hint is set it will automatically use that rendering hint.
